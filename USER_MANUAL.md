## 1. Обзор

**Script Screen** — инструмент для написания и структурирования сценариев, построенный вокруг концепции визуальных карточек (карточек сцен). Каждая сцена представлена в виде перетаскиваемой карточки на холсте. Карточки связаны с действующими лицами, местами, деталями и временными периодами, что позволяет видеть общую структуру сценария и одновременно писать текст сцен в полноценном текстовом редакторе.

**Основные понятия:**

| Термин | Значение |
|---|---|
| **Проект** | Контейнер верхнего уровня, содержащий все данные сценария (название, версия, авторы) |
| **Сценарий** | Непосредственно сценарий — титульный лист, логлайн, синопсис и текст сцен |
| **Карточка (сцена)** | Одна сцена: порядковый номер, заголовок, описание и теги атрибутов |
| **Действующее лицо** | Персонаж или человек, появляющийся в сценарии |
| **Место** | Место, где происходит действие сцены |
| **Деталь** | Сюжетный элемент, мотив или предмет, связывающий сцены |
| **Время действия** | Временной маркер (например, «Утро», «Три дня спустя») |

## 2. Начало работы

### Создание нового проекта

1. Запустите приложение. По умолчанию открывается панель **Проект**.
2. Откройте меню (правый верхний угол) и выберите **Новый проект**.
3. В диалоговом окне введите:
   - **Название** — только буквы, цифры, символы `_`, `-` и `.`.
   - **Версия** — строка версии проекта (например, `1.0`).
4. Подтвердите. Новый проект создан и готов к работе.

### Открытие существующего проекта

1. В меню панели **Проект** выберите **Открыть проект**.
2. Найдите и выберите файл проекта `.json`.
3. Проект загружается вместе со всеми карточками, действующими лицами, местами, деталями и временными периодами.

> **Совет:** Приложение запоминает последний открытый проект и автоматически загружает его при следующем запуске.

### Сохранение

Проект сохраняется автоматически через настраиваемый интервал (автосохранение). Сохранение также запускается при любой операции с проектом (создание, открытие, экспорт). Кнопка «Сохранить» в процессе обычного редактирования не требуется.

## 4. Управление проектом

Доступно из панели **Проект**.

### Поля

| Поле | Описание |
|---|---|
| **Название** | Уникальный идентификатор проекта (буквы, цифры, `_`, `-`, `.`) |
| **Версия** | Строка версии проекта |
| **Язык** | Язык, используемый при транскрипции аудио |
| **Автор(ы)** | Одно или несколько имён авторов, связанных с проектом |

### Действия в меню

- **Новый проект** — создать новый пустой проект (текущие данные сохраняются перед очисткой).
- **Открыть проект** — загрузить проект из файла `.json`.
- **Редактировать проект** — изменить название, версию, язык или авторов.
- **Экспорт проекта** — экспортировать сценарий в формат PDF.
- **Выход** — сохранить и закрыть приложение.

## 5. Сводка сценария

Доступно из панели **Сценарий**.

Этот раздел содержит вводные данные сценария — информацию, которая отображается перед сценами.

| Поле | Описание |
|---|---|
| **Название** | Название сценария (отображается на титульном листе) |
| **Автор** | Имя автора на титульном листе (наследуется от авторов проекта) |
| **Дата** | Дата на титульном листе |
| **Место** | Место производства или город |
| **Логлайн** | Краткая аннотация к сценарию в одно предложение (≈ 25 слов) |
| **Синопсис** | Развёрнутое изложение сценария (до ~5 страниц) |

Откройте меню и выберите **Редактировать**, чтобы изменить любое из этих полей.

## 6. Карточки (сцены)

Доступно из панели **Карточки**. Это основная рабочая панель.

### Холст

На холсте все сцены отображаются в виде перетаскиваемых карточек, расположенных в сетке. Карточки можно свободно перемещать, создавая визуальную структуру, отражающую повествование сценария.

### Структура карточки

Каждая карточка содержит:
- **Порядковый номер** — позиция сцены в сценарии (редактируется).
- **Заголовок / Описание** — краткое описание сцены.
- **Теги атрибутов** — действующие лица, места, детали и временные периоды, привязанные к сцене.

### Создание карточки

**Shift + Click** в любом пустом месте холста создаёт новую карточку сцены в этой позиции.

Справа (или ниже холста) открывается форма, в которой нужно заполнить:

| Поле | Описание |
|---|---|
| **Номер** | Номер сцены (целое положительное число ≥ 1). Определяет порядок сцен в сценарии. |
| **Заголовок** | Краткое название сцены. |
| **Описание** | Краткое описание, отображаемое на карточке. |
| **Действующие лица** | Персонажи, участвующие в этой сцене (множественный выбор). |
| **Места** | Где происходит действие сцены (множественный выбор). |
| **Детали** | Сюжетные элементы или мотивы в этой сцене (множественный выбор). |
| **Время действия** | Временной период сцены (множественный выбор). |

### Редактирование карточки

- Нажмите **иконку карандаша** на карточке, чтобы открыть форму редактирования, **или**
- Нажмите на карточку, чтобы выделить её — форма появится автоматически.

Внесите изменения в форму. Изменения сохраняются при переходе к другой карточке или снятии выделения.

### Удаление карточки

Нажмите **иконку X** на карточке. Карточка и связанный с ней файл содержимого сцены удаляются безвозвратно.

### Перемещение и изменение размера карточек

- **Переместить**: перетащите карточку за заголовок или тело.
- **Изменить размер**: перетащите угловой маркер в правом нижнем углу карточки.

### Выделение и снятие выделения

- **Выделить**: нажмите на карточку. Её рамка станет **розовой**, содержимое загрузится в редактор.
- **Снять выделение**: нажмите на выделенную карточку ещё раз. Рамка вернётся к стандартному цвету, содержимое редактора сохранится и очистится.

Одновременно может быть выделена только одна карточка.

### Транскрипция аудио

В меню панели **Карточки** выберите **Транскрибировать аудиофайл**, чтобы преобразовать аудиофайл (MP3, WAV, M4A) в текст и поместить его в редактор выбранной сцены. Для транскрипции используется языковая настройка проекта.

## 7. Действующие лица

Доступно из панели **Действующие лица**.

### Добавление действующего лица

1. Откройте меню и выберите **Добавить действующее лицо**.
2. Заполните поля:
   - **Имя** (обязательно) — имя персонажа.
   - **Описание** (необязательно) — заметки о персонаже.
3. Подтвердите.

### Редактирование

Нажмите на действующее лицо в списке, чтобы открыть форму редактирования. Внесите изменения и подтвердите.

### Удаление

Откройте форму редактирования действующего лица и нажмите кнопку **Удалить**.

> Удаление действующего лица снимает его со всех карточек сцен, к которым оно было привязано.

Доступно из панели **Детали**.

Детали — это повторяющиеся сюжетные элементы, мотивы, предметы или нарративные нити, связывающие сцены между собой.

Нажмите на элемент списка для редактирования или удаления.

## 11. Редактор сценария

Редактор — это полноценный текстовый редактор (TinyMCE), открывающийся в отдельном окне рядом с холстом.

### Показать и скрыть редактор

В меню панели **Карточки** выберите **Показать редактор** или **Скрыть редактор**, чтобы переключить видимость окна редактора.

### Принцип работы редактора

- При **выделении** карточки сцены её содержимое загружается в редактор.
- Пишите и форматируйте текст сцены в редакторе.
- При **выделении другой карточки** текущее содержимое **автоматически сохраняется** перед загрузкой новой сцены.
- При **снятии выделения** с карточки содержимое сохраняется и редактор очищается.
- При **удалении** карточки связанный файл содержимого удаляется безвозвратно.

### Форматирование

Редактор TinyMCE поддерживает стандартное форматирование текста:

- **Жирный**, *курсив*, подчёркивание
- Заголовки (H1–H6)
- Маркированные и нумерованные списки
- Стили абзацев

## 12. Фильтрация

На холсте **Карточки** можно фильтровать отображаемые карточки, нажимая на теги атрибутов.

### Как использовать фильтр

1. Найдите теги атрибутов в верхней части панели **Карточки** или на любой карточке.
2. Нажмите на тег (действующее лицо, место, деталь или время действия), чтобы активировать фильтр.
3. На холсте останутся видны только карточки, **помеченные этим атрибутом**.
4. Нажмите на тег повторно, чтобы снять фильтр.

Одновременно могут быть активны несколько фильтров. Карточка отображается, если она соответствует **хотя бы одному** из активных фильтров.

## 14. Хранение данных

### Формат файла

Проекты хранятся в виде **JSON-файлов** (`.json`). Каждый файл проекта содержит все метаданные: карточки, действующие лица, места, детали, временные периоды и сводку сценария.

### Содержимое сцен

Текст каждой сцены хранится в отдельном **HTML-файле** в папке проекта. Эти файлы создаются и управляются автоматически.

### Папка проекта

При создании или версионировании проекта создаётся папка с временно́й меткой, в которой хранятся все связанные файлы. Не переименовывайте и не перемещайте эти файлы вручную.

### Правила проверки данных

| Поле | Правило |
|---|---|
| Название проекта / сценария | Только буквы, цифры, `_`, `-`, `.` |
| Номер карточки | Целое положительное число ≥ 1 |
| Поля «Название» (лица, места и т.д.) | Не может быть пустым |
| Поля «Описание» | Необязательны |
