1. Термины
	1.1 Сценарий
		Титульный лист
		Логлайн - краткая аннотация к сценарию, передающая основную его суть (25 слов)
		Синопсис - краткое изложение сценария (до 5 страниц)
		Версия
		Сцена - составная часть сценария, характеризуется единством времени и места действия, действующих лиц
		Эпизод - логическая последовательность сцен
	1.2 Действующие лица
		Описание
	1.3 Карточка (ментальная) - краткое описание сцены с указанием времени, места и действующих лиц. Последовательность карточек задает последовательность повествования.
		Нумерация сквозная.
	1.4 Деталь - выразительная подробность, с помощью которой создается художественный образ. Может воспроизводить черты внешности, особенности одежды, 
		обстановки, предмета, переживания или поступка. Деталь может формировать связь между карточками
	1.5 Сюжетная линия последовательность карточек, объединенных единством действующих лиц

5. Защита.
5.1 Программа может быть запущена без идентификационного файла. Но в этом случае её функциональность ограничена: невозможно конвертировать в pdf.
5.2 Для получения полного доступа к функциональности программы нужно купить идентификационный файл (для платформ window, macos, для linux бесплатно)
5.3 При всяком запуске без идентификационного файла программа получает что-то одно из 4.1, 4.2, 4.3. На основании полученного она генерирует хэш, 
который и нужно передать производителю программы.
5.4 Пользователь передает хэш производителю
5.5 Получив хэш, производитель генерирует пару ключей приватный и публичный.
5.6 Производитель создает идентификационный файл, содержащий хэш и публичный ключ и подписывает его эцп.
5.7 Производитель передает идентификационный файл и эцп пользователю после получения оплаты
5.8 При последующих запусках, программа проверяет идентификационный файл с помощью эцп. При положительном результате проверке, сравнивает хэш из файла с 
хэшем чего-то из 4.1, 4.2, 4.3. При совпадении хотя бы с одним из хэшей 4.1, 4.2, 4.3 включает полную функциональность.

6. БД пользователей
